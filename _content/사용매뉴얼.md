# 쿡더디자인 블로그 → GitHub 백업 파이프라인 사용 매뉴얼

네이버 블로그 **cookthedesign**(`blog.naver.com/cookthedesign`)의 공개글 **666개**를
스크랩하여 본문·이미지를 가공한 뒤 GitHub 레포 **`papascompany/Printguide`** 에 백업하고,
카테고리 구조대로 볼 수 있는 링크 목록을 만드는 도구 모음입니다.

> 자매 프로젝트: `../blogposting` (jumbocard → Blogger 이전). 구조는 비슷하나 별개 폴더입니다.

## 2. 파이프라인

```
fetch_list.py     카테고리 구조 + 전체 글목록 수집
      │            → data/categories.json, data/postlist.json
      ▼
scraper.py        각 글 본문 추출 + 브랜드 치환 + 푸터 제거 (이미지 URL만 수집)
      │            → data/posts/{logNo}.json
      ▼
download_media.py 콘텐츠 이미지 다운로드 (푸터 제외분만)
      │            → data/images/{logNo}/  + 각 posts JSON 에 image_local 기록
      ▼
(git push)        data/images/* → papascompany/Printguide → 커밋 SHA 확보
      ▼
make_cdn_map.py   커밋 SHA 기준 CDN URL 매핑
      │            → data/blob_map.json
      ▼
build.py          본문 이미지 URL을 jsDelivr CDN URL로 치환 + 미리보기 HTML
      │            → data/ready/{logNo}.json, data/preview/{logNo}.html
      ▼
build_index.py    카테고리 구조대로 링크 목록
                   → INDEX.md, index.html
```

이미지를 GitHub+jsDelivr로 재호스팅하는 이유: **네이버 이미지는 핫링크(외부 임베드) 차단**이라
원본 URL을 그대로 쓰면 깨집니다. 공개 레포에 올리고 커밋 SHA를 고정한 CDN URL은 영구 불변입니다.

## 3. 실행 방법

```bash
# 0) 의존성 (blogposting 의 .venv 를 심볼릭 링크로 공유)
#    requests, beautifulsoup4, lxml

# 1) 카테고리 + 글목록
.venv/bin/python fetch_list.py

# 2) 본문 스크랩 (브랜드 치환 + 푸터 제거)
.venv/bin/python scraper.py
#    특정 개수만 테스트: .venv/bin/python scraper.py --sample 10

# 3) 이미지 다운로드 (재실행 시 받은 것은 건너뜀)
.venv/bin/python download_media.py

# 4) GitHub 푸시 후 SHA 확보
cd data/images
git init -b main && git add -A && git commit -m "images"
git remote add origin https://github.com/papascompany/Printguide.git
git push -u origin main
SHA=$(git rev-parse HEAD); cd ../..

# 5) CDN 매핑 + 본문 URL 치환 + 미리보기
.venv/bin/python make_cdn_map.py "$SHA"
.venv/bin/python build.py

# 6) 카테고리 링크 목록
.venv/bin/python build_index.py
```

## 4. 푸터 배너 제거 규칙 (중요)

푸터 배너는 **모든 마케팅 글의 맨 끝**에, `se-talktalk` 위젯("궁금할 땐 네이버 톡톡하세요!")
**바로 뒤**에 1~3장씩 붙는 홍보 이미지입니다(전화연결·홈페이지 바로가기 등).

판정 규칙(`scraper.py`):
- **마지막 `se-talktalk` 뒤가 전부 이미지/빈줄/구분선일 때만** 그 이미지들을 푸터로 보고 제거.
- talktalk 뒤에 실제 텍스트가 있으면 → 그 talktalk은 본문 중간이므로 **제거하지 않음**(콘텐츠 보존).
- talktalk이 없으면 → 아무것도 지우지 않음(옛 기획/문서 글의 끝 이미지는 콘텐츠).

이 규칙은 ① 정상 글의 푸터 1~3장 정확 제거, ② 캡션이 인용구(`se-quotation`)인 이미지 모음글,
③ talktalk이 글 앞쪽에 있는 글에서 **콘텐츠 이미지를 오삭제하지 않도록** 검증되었습니다.

## 6. 워드프레스로 이전 (WXR)

`make_wxr.py` 로 워드프레스 가져오기 파일을 생성합니다. 본문은 **네이버 의존 0**
(텍스트·이미지 모두 분리, 본문 속 네이버 링크/URL 제거 완료), 원본 발행일·카테고리 계층 보존.

```bash
.venv/bin/python make_wxr.py                  # 단일 파일(27MB) — WP-CLI 가져오기용
.venv/bin/python make_wxr.py --split 120      # 분할 6개(~4~6MB) — 웹 업로드 한도 안전
.venv/bin/python make_wxr.py --draft          # 초안 상태로
.venv/bin/python make_wxr.py --image-base https://media.example.com/print  # 이미지 호스트 변경 시
```

### 가져오기 절차 (이미지 = CDN 유지 + WP 내재화 방식)
1. 워드프레스 관리자 → **도구 → 가져오기 → WordPress** (Importer 플러그인 설치).
2. 분할 파일을 **part01 → part06 순서로** 업로드(웹 업로드 한도 회피). 단일 27MB는 WP-CLI:
   `wp import cookthedesign.wordpress.xml --authors=create`
3. 가져온 직후 본문 이미지는 **jsDelivr CDN URL**(Printguide 공개 상태에서 표시됨).
4. **이미지 내재화**: *Auto Upload Images* 또는 *Import External Images* 플러그인 실행 →
   본문의 외부 이미지를 워드프레스 미디어로 흡수. 이후 본문 URL이 WP 자체 URL로 바뀜.
5. 내재화 완료 후에는 **Printguide 레포를 비공개로 전환해도** 이미지가 정상 표시됩니다.

> 이미지를 처음부터 WP 미디어/별도 스토리지에 둘 거면, 그 URL로 `--image-base` 재생성하면 됩니다.

## 7. 한계 / 주의

- **이미지 안에 그려진 글자**(로고/배너 텍스트)는 이미지라 브랜드 치환이 불가합니다.
  (브랜드 텍스트가 가장 많은 푸터 배너는 어차피 제거됩니다.)
- 비공개(비밀) 글은 공개 스크래핑으로 잡히지 않습니다.
- jsDelivr CDN은 **공개 레포만** 서빙하므로 Printguide는 public 입니다.
- 네이버 글목록 API는 제목에 백슬래시가 있으면 표준 JSON 파싱이 깨져 lenient 파싱을 씁니다.
- 본문은 스마트에디터(SmartEditor ONE) HTML을 보존합니다. 네이버 전용 CSS는 무시되지만
  텍스트·이미지·구조는 정상 표시됩니다.
